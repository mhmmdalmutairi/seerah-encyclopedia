# موسوعة السيرة النبوية المؤسسية

> رصد وتشخيص الكيانات (المؤسسات، الأفراد، المنشورات، المنابر) المتخصِّصة بخدمة سيرة النبي محمد ﷺ على مستوى العالم.

🌐 **بوابة الموسوعة**: https://mhmmdalmutairi.github.io/seerah-encyclopedia/

---

## ما هذا؟

موسوعة بيانات منظَّمة، عربية الواجهة، مفتوحة المصدر، تحوي **436 كيانًا جوهريًّا** متخصِّصًا بالسيرة النبوية في **52 دولة**، مع طبقة سجلّ ثانوي (34 كيانًا) وعلامتَي فجوات موثَّقة. كلّ كيان بطاقة مفصَّلة بـ35 حقلًا (مَنشأ، نوع، إقليم، تخصُّص، مدرسة منهجية، جمهور مستهدف، إلخ).

**معيار الإدراج الصارم**: الكيان مهمَّته الأساسية حياة النبي محمد ﷺ تحديدًا (السيرة، المغازي، الشمائل، دلائل النبوة، المولد، جغرافيا السيرة، الطب النبوي، السيرة في القرآن). تُستبعَد المؤسسات الدينية العامّة بلا برنامج سيرة معلَن.

## المنتج

- **بوابة تفاعلية** (`index.html`): 10 تبويبات — تشخيص، كيانات (بحث + فلترة + فرز)، فجوات، فرص، شبكة علاقات، إحصاءات، منهجية، مراجع، تصدير، صحّة بيانات
- **API بسيط** (`api.html` + `data.json`): اقرأ كلّ البيانات كملفّ JSON واحد، أو استدعِ الكيانات والإحصاءات والفجوات برمجيًّا
- **تغذية (`feed.json`)** و**خريطة موقع (`sitemap.xml`)** للأرشفة
- **منهجية معلَنة** ([`METHODOLOGY.md`](METHODOLOGY.md)): معايير الإدراج/الاستبعاد، مستويات التحقُّق، التحفُّظات
- **تدقيق آلي** ([`tools/validate.py`](tools/validate.py)): يَمنع أيّ commit يكسر المخطّط — يعمل عبر pre-commit hook + GitHub Actions
- **مسح صحّة شهري** ([`tools/quality_scan.py`](tools/quality_scan.py)): اكتمال الحقول، صحّة الروابط، الفجوات، درجات الجودة → تبويب «صحّة البيانات»

## طبقة المشروع التطبيقي (اختيارية)

الموسوعة منتج عالمي قائم بذاته. لكن البنية تَدعم أيضًا «طبقة مشروع تطبيقي» — توصيفات شراكة لمشروع معيَّن يستفيد من البيانات. مفصولة في `data/project_annotations.json`.

لتفعيل الطبقة في الواجهة:
```bash
python3 tools/sheikh_mode.py on   # يُظهر تبويب «المشروع الجامع» + شريط أدوات «عرض المشروع»
python3 tools/sheikh_mode.py off  # الوضع العامّ (افتراضي)
```
أو عبر URL: `?sheikh=1` ، أو `localStorage.setItem('sheikh_mode','1')`.

## المساهمة

اقتراح إضافة كيان أو ملاحظة بيانية: عبر [`suggest.html`](suggest.html) في الموقع.

كلّ commit يَجتاز:
1. `tools/validate.py` — يَتحقّق من المخطّط (مفاتيح، أنواع، قِيم enum، تفرُّد المعرّفات، صحّة العلاقات)
2. `tools/quality_scan.py --no-network` (في GitHub Actions) — يضمن المسح الدوري لا يكسر الصياغة
